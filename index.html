GoDaddy DNS Setup Guide

Step-by-step instructions for connecting your GoDaddy domain to GitHub Pages

DNS Records Explained

When setting up DNS records on GoDaddy, you need to understand what each field means:

Field Purpose Example Values
Type Specifies the kind of DNS record A, CNAME, MX, TXT
Name The hostname or domain name @, www, mail
Value The destination address or value IP address or domain name
TTL Time to Live - how long DNS servers should cache the record 1 hour, 30 minutes

Note: For GitHub Pages, you'll primarily use A records for the root domain (@) and a CNAME record for the www subdomain.

GitHub Pages DNS Configuration

To connect your GoDaddy domain to GitHub Pages, you need to create the following DNS records:

Type Name Value TTL
A @ 185.199.108.153 1 hour
A @ 185.199.109.153 1 hour
A @ 185.199.110.153 1 hour
A @ 185.199.111.153 1 hour
CNAME www yourusername.github.io 1 hour

Important: Replace "yourusername" with your actual GitHub username in the CNAME record value.

Step-by-Step Guide for GoDaddy
1
Log in to your GoDaddy account

Go to godaddy.com and sign in to your account.

2
Access your DNS management page

Navigate to your Domain Portfolio and click on the domain you want to manage (jeyrob.com). Then click on the "DNS" button to manage your DNS records.

3
Delete existing A records

Remove any existing A records for your domain to avoid conflicts.

4
Add new A records

Click "Add" to create new records. For each of the four GitHub IP addresses, create an A record with:

  • Type: A
  • Name: @
  • Value: One of the four GitHub IP addresses (185.199.108.153, 185.199.109.153, etc.)
  • TTL: 1 hour
5
Add a CNAME record for www

Create a CNAME record with:

  • Type: CNAME
  • Name: www
  • Value: yourusername.github.io (replace with your GitHub username)
  • TTL: 1 hour
6
Save your changes

Click "Save" to apply your DNS changes. It may take up to 48 hours for DNS changes to propagate globally.

7
Configure GitHub Pages

In your GitHub repository settings, go to the Pages section and add your custom domain (jeyrob.com). Check the "Enforce HTTPS" option once it becomes available.
